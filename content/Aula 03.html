A linguagem Python

De acordo com a Wikipédia:

Python é uma linguagem de programação de alto nível, interpretada de script, imperativa, orientada a objetos, funcional, de tipagem dinâmica e forte. Foi lançada por Guido van Rossum em 1991. Atualmente, possui um modelo de desenvolvimento comunitário, aberto e gerenciado pela organização sem fins lucrativos Python Software Foundation. Apesar de várias partes da linguagem possuírem padrões e especificações formais, a linguagem, como um todo, não é formalmente especificada. O padrão de facto é a implementação CPython. A linguagem foi projetada com a filosofia de enfatizar a importância do esforço do programador sobre o esforço computacional. Prioriza a legibilidade do código sobre a velocidade ou expressividade. Combina uma sintaxe concisa e clara com os recursos poderosos de sua biblioteca padrão e por módulos e frameworks desenvolvidos por terceiros.

Os principais motivos para se aprender Python são:

  • Sintaxe simples

  • Melhor uso do tempo do programador

  • Versatilidade

  • Grande suporte para ciência de dados

  • Mercado amplo

  • Aceitação de mercado

No nosso caso, Python foi a escolha pelo suporte existente à ciência de dados - com um ecossistema de bibliotecas que permite um fluxo de trabalho de ponta a ponta feito em Python*. Ainda assim, há outras linguagens de programação comumente usadas em ciência de dados, principalmente:

  • R: Originalmente surgida de contexto de estatística e análise de dados, tem uso ainda mais especializado em análise que Python, mas por outro lado, frequentemente não dispõe de bibliotecas genéricas fora da área de estatística/ciência de dados/etc.

  • Julia: Uma linguagem mais nova (2012), com o objetivo de ser “rápida como C, dinâmica como Ruby, fácil para estatística como R, genérica como Python, …” e em alto crescimento.

Diversas outras linguagens são usadas em ciência de dados (SQL, etc.). Citamos as duas acima por que são as mais próximas em termo de comparação.

Operando um notebook Jupyter

Ao usar um notebook, há dois modos - comando e edição. O modo comando permite navegar (subir e descer entre as células), apagar / criar células, executar a célula, etc.

O modo de edição é que você faz alterações internas na célula (escreve código, Markdown, etc.):

  • shift + enter: executa a célula atual, e seleciona a célula seguinte.

  • ctrl + enter: executa a célula atual

  • alt + enter: executa a célula atual, insere nova célula abaixo.

  • ctrl + S: salva o notebook

Para usar o modo comando, aperte Esc (a célula fica com borda azul ). Para mudar para edição, aperte Enter (a borda fica verde ).

Em modo de edição:

  • ↑: navega para cima

  • ↓: navega para baixo

  • shift + ↑: seleciona células para cima

  • shift + ↓: seleciona células para cima

  • A: insere célula acima

  • B: insere célula abaixo

  • DD (apertar D duas vezes): apaga a célula selecionada

  • II (apertar I duas vezes): interrompe o kernel (caso você faça um laço eterno, por exemplo)

  • Z: desfaz operação

  • tab: autocompletar / indentar

  • L: numera as linhas

Para mudar de código para Markdown (e vice-versa) - em modo de edição (ESC):

  • M: muda para Markdown

  • Y: muda para código

Você pode consultar os atalhos existentes no modo de comando (ESC):

  • P: abre a barra de pesquisa.

  • H: lista todos os atalhos

Além disto, você pode criar atalhos personalizados Help > Edit Keyboard Shortcuts. Exemplo: um comando que eu uso com frequência é restart the kernel and re-run the whole notebook (⏩ no menu), que eu salvei como ctrl + R.

O que são variáveis?

A ideia básica de programação é manipular (fazer todo tipo de operação) em valores (números, texto, etc). E o componente básico dessa manipulação são as variáveis.

Vamos começar com uma analogia - imagine que você trabalha numa loja de peças automotivas gerenciando o estoque de peças. As peças estão num depósito no fundo da loja empacotadas em caixas. Há diversos tipos e tamanhos de caixas, e cada caixa pode caber uma ou mais peças - inclusive de tipos e tamanhos diferentes. Aqui temos os dois elementos principais: o conteúdo que vai em cada caixa (isto é, os valores) e cada formato de caixa com suas características (os tipos).

Exemplo: em Python, você pode criar uma variável ao definir um nome e associar a um valor: ano = 365

Neste exemplo vimos claramente o valor (365) e o nome da variável (ano). Onde então está o tipo?

Essa noção é importante porque há linguagens de programação que são explicitamente tipadas (C++ por exemplo) e outras que são dinamicamente tipadas como Python e R - Julia por outro lado é opcionalmente tipada.

O que isto quer dizer é que em algumas linguagens de programação você precisa deixar claro o tipo de valores podem ser contidos na variável durante sua declaração (i.e., tipo nome = valor). Veja um exemplo em C++:

Exemplo: int ano = 365;

Isto quer dizer que no paradigma explícito você precisa informar três coisas - o nome da variável (ano), o tipo que ela recebe (números inteiros int) e seu valor (ou conteúdo - 365). Isto é, o tipo foi explicitamente definido durante a declaração da variável (daí a expressão “explicitamente tipada”).

Python, por outro lado, é dinamicamente tipada. Portanto, não é necessário informar ao interpretador que tipo de dados contém cada variável. Como funciona então? Nos bastidores, o próprio Python “lê” o conteúdo e infere seu tipo).


Exemplo: Se você mudar em C++, ano = "2021";, retornará um erro, porque ano só recebe números inteiros, e não texto (valores entre aspas, por exemplo). Em Python, por outro lado, é perfeitamente aceitável (e também perigoso em certas situações) mudar de ano = 365 para ano = "2021".

Dica: Em Python chamamos esse procedimento de moldagem implícita (implicit casting) porque o interpretador escolhe para nós o tipo de dados (e não nós, o que seria explícito).

A vantagem da abordagem dinâmica é que requer menos trabalho e planejamento por parte do programador. Mas há desvantagens nesse tipo de abordagem:

  • Faz com que os programas se tornem mais lentos (é um passo a mais para o interpretador executar).

  • Quando você declara o tipo da variável previamente, você faz uma “reserva” de memória. Em casos mais adiante vamos ver que isso faz com que a “reserva” seja maior do que o necessário, fazendo com que um conjunto de dados ocupe mais espaço que o necessário.

  • Um problema adicional é que pode haver erros na interpretação do tipo de dado levando a problemas de análise de dados feita de forma incorreta

Voltando à analogia da loja de autopeças. Podemos delimitar uma série de variáveis sobre uma peça:

  • Nome

  • Marca

  • Peso

  • Preço

  • Código

  • Quantidade

Perceba que cada variável a respeito de uma peça tem características intrínsecas: “nome” e “marca” serão texto, “quantidade” será um número inteiro (5 ou 6 peças, mas não 3,7 peças), “peso” e “preço” serão fracionados (8,9 kg; R$ 240,76) e assim por diante.

Cada linguagem de programação difere em como tratar os tipos de variáveis. Vamos ver em seguida como Python lida com esse assunto.

Dica: Se você planeja coletar dados em formato tabular, cada coluna terá um nome semelhante à variável. Crie um arquivo separado chamado de “dicionário de variáveis”. Escreva o nome da variável como se encontra no quadro de dados e sua definição - o que é, qual a medida, detalhes, etc. Isso facilita o trabalho de quem for futuramente rever seus dados (inclusive você mesmo depois de muito tempo)

Dica: A explicação de variável aqui serve para os propósitos de um curso cujo foco é análise de dados, já que não vamos precisar ficar olhando debaixo do capô. No entanto, vale a pena observar que o que chamamos de variável em Python não é uma variável no sentido explícito e é bastante diferente de C por exemplo. Não vamos entrar em detalhes aqui mas em Python um objeto não é “dono” de um bloco de memória e sim uma palavra atrelada a um bloco de memória (ponteiro). Isso afeta diversas operações como vamos ver ao longo do curso. Uma discussão interessante sobre o assunto pode ser lida aqui.

Declarando variáveis

Vimos um exemplo prévio de declaração de variáveis em Python, mas há alguns detalhes importantes. Primeiro, Python é sensível a tipos de letras (case sensitive) - isto quer dizer que ano, Ano, ANO e AnO serão interpretados como coisas diferentes.

Segundo, a ordem importa - primeiro vem o nome, o sinal de = (que indica “recebe”) e o valor pretendido.

Dica: O símbolo de = só é usado para atrelar valores a variáveis e por isso = NÃO é usado para teste de igualdade. Mais adiante você verá que para fazer testes de igualdade Python usa ==.

Em Python você terá um erro de sintaxe se tentar fazer o contrário (365 = ano).

# Leia: ano recebe 365
ano = 365
ano

365

Dica: O símbolo = nessa expressão equivale a recebe ou contém

# erro
# 365 = ano

Compare com R em que você pode fazer em ambas as direções ( ano <- 365, 365 -> ano, ano = 365 ou 365 = ano).

Python tem um recurso chamado unpacking (desempacotamento) que serve para declarar mais de uma variável ao mesmo tempo (contanto que a quantidade de valores e de variáveis seja igual). Uma desvantagem de fazer isso é na hora de alterar o código, legibilidade e organização.

Exemplo:

ano, mes, dia, ocasiao, venda = 365, 12, 25, "natal", 55.89

Nomeando variáveis

Python limita a forma como você nomeia uma variável. Primeiro, você não pode nomear uma variável com umas das palavras protegidas da linguagem (keywords).

Vamos verificar as palavras protegidas do Python:

import keyword
print(keyword.kwlist)

['False', 'None', 'True', '__peg_parser__', 'and', 'as', 'assert', 'async', 'await', 'break', 'class', 'continue', 'def', 'del', 'elif', 'else', 'except', 'finally', 'for', 'from', 'global', 'if', 'import', 'in', 'is', 'lambda', 'nonlocal', 'not', 'or', 'pass', 'raise', 'return', 'try', 'while', 'with', 'yield']

Como exemplo, vamos criar uma variável com um texto igual a uma das palavras protegidas, e vamos testar se está na lista:

nome_da_variavel = "while"
nome_da_variavel in keyword.kwlist

True

# erro
# nome_da_variavel_2 = while

Segundo, há outras restrições de nomeação de variáveis:

  • Pode ter qualquer tamanho (mas use seu bom senso)

  • Pode conter letras maiúsculas (“A, B, …”) e minúsculas (“a, b, …”)

  • Pode conter underline (_)

  • Pode conter dígitos (0, 1, 2…) - mas não pode iniciar com dígitos

Nomes de variáveis válidos: nomeDoCliente, mêsdonascimento, receita2022, etc.

No entanto, a equipe mantenedora do Python edita regularmente guias da linguagem e um desses (PEP-8) define algumas regras que ajudam na homogeneização da escrita em Python. Sugiro que você leia essas regras depois, já que há regra para cada tipo de coisa (variável, função, classe, etc).

Para variáveis, sugere-se a convenção snake case:

  • Usar letras minúsculas

  • Separar as palavras (caso haja mais de uma) com um _

Assim, os nomes de variáveis acima ficariam: nome_do_cliente, mês_do_nascimento, receita_2022,

Outras sugestões são retirar palavras como “do”, “da”, “para”, etc., entre palavras de uma variável e criar variáveis sem acento ou sinais (o que pode dar erro de codificação dependendo do caso) - portanto mês_do_nascimento ficaria melhor como mes_nascimento.

Dica: A exceção são as constantes - isto é, valores que você não pretende alterar durante a execução do código. Sugere-se que as constantes sejam em letras maiúsculas, portanto PI = 3.14159.

Dica: Quando for preparar dados tabulares ou planejar uma coleta de dados, nomeie as colunas com essas convenções acima. Isso facilitará muito o trabalho de limpeza nos nomes futuramente.

Ordem das operações

É importante lembrar que Python segue a ordem das operações PEMDAS, ou seja, avalia e executa na seguinte ordem:

  1. Parênteses

  2. Exponenciação

  3. Multiplicação e Divisão

  4. Adição e Subtração

No caso de multiplicação e divisão que estão no mesmo nível (da mesma forma que adição e subtração), Python avalia e executa da esquerda para a direita.

mes = 4
ano = 12

decimo_terceiro = 365 - (semana * mes * ano)

Texto (strings)

Subagrupamento por fatiamento (slicing)¶

Funciona com objetos iteráveis (string, lista, etc). A lógica é colocar [] após o objeto e passar alguns parâmetros que auxiliem a retornar o que se quer:

  • O primeiro parâmetro define o início (ou só o que retorna, se for um só).

  • O segundo parâmetro (opcional) indica o fim, lembrando que em fatiamento Python não inclui o segundo número. Portanto, indica fim-1.

  • O terceiro parâmetro (opcional) indica a distância ou pulo entre os elementos (de 1 em 1, de 2 em 2, etc)

  • A ordem de retorno é sempre da esquerda para a direita (do obj[0] até o obj[-1]), mesmo que você use números negativos (texto[-7:-3]):

"Habib's"

esfirra_do_habibs[0:18:2]

"efrad ai'"

esfirra_do_habibs[::2]

"efrad ai'"

Dica: Para facilitar, você pode abreviar a forma de escrever fatiamento:

  • string[número] retorna sempre o elemento desejado

  • string[número:] retorna do elemento desejado até o último

  • string[:número] retorna do primeiro até o número-1

  • string[:] retorna todos os elementos (mas é igual a string)

  • string[::número] retorna todos os elementos com número de distância entre eles

Estilo str.format()

No estilo str.format(), você cria um texto como antes, usando aspas simples ou duplas. O pulo do gato é passar um método chamado .format() que permite você manipular as variáveis que aparecem no texto, por fora. Como se faz? O lugar onde a variável deve ficar no texto é substituída por {}, e a referência à variável é passada como parâmetro do método .format():

# primeira versão
"Um ano tem", ano, "dias!"

# segunda versão com str.format()
"Um ano tem {} dias!".format(ano)

'Um ano tem 365 dias!'

Se houver mais de uma variável, basta com que a ordem seja obedecida para o interpretador saber qual par de colchetes equivale a qual variável:

"Um ano tem {} dias, e cada mês em média tem {}!".format(ano, ano / 12)

'Um ano tem 365 dias, e cada mês em média tem 30.416666666666668!'

À primeira vista pode parecer que dá mais trabalho, mas é mais reproduzível e mais fácil de mudar depois. Uma vantagem de usar .format() é que você pode referenciar as variáveis fora de ordem, indicando dentro dos {} seu nome:

mes = ano / 12

"Um ano tem {ano} dias, e cada mês em média tem {mes}!".format(mes=mes, ano=ano)

'Um ano tem 365 dias, e cada mês em média tem 30.416666666666668!'

Estilo f"string

O estidlo f"string" é similar ao anterior (str.format()). A grande diferença é que você já informa ao interpretador que o texto tem alguma formatação prefixando o texto com f" .... " e passa a variável como argumento dentro dos {}:

# primeira versão intercalada
"Um ano tem", ano, "dias!"

# segunda versão com str.format()
"Um ano tem {} dias!".format(ano)

# terceira versão com f-string:
f"Um ano tem {ano} dias!"

'Um ano tem 365 dias!'

Da mesma forma, você pode fazer operações dentro dos {}:

# segunda versão com str.format()
"Um ano tem {} dias, e cada mês em média tem {}!".format(ano, ano / 12)

# terceira versão com f-string:
f"Um ano tem {ano} dias, e cada mês em média tem {ano / 12} dias!"

'Um ano tem 365 dias, e cada mês em média tem 30.416666666666668 dias!'

Você pode formatar como uma variável é apresentada entre colchetes. Um exemplo é o caso acima, em que fizemos a operação ano / 12 - o resultado é preciso, mas apresenta muitas casas decimais. Você até poderia usar a função nativa round() para arredondar, mas há uma forma interessante de fazer isso direto nos colchetes: {variável:.Xf} em que X é o número de casas decimais a apresentar. Exemplo:

# sem delimitar casas:
print(f"Um ano tem {ano} dias, e cada mês em média tem {ano / 12} dias!")

# delimitando casos:
print(f"Um ano tem {ano} dias, e cada mês em média tem {ano / 12:.1f} dias!")

Um ano tem 365 dias, e cada mês em média tem 30.416666666666668 dias!
Um ano tem 365 dias, e cada mês em média tem 30.4 dias!

Funções

Aos poucos já vimos diversas funções em Python ao longo do curso e chegou a hora de vermos as funções com mais detalhes. Python tem três tipos básicos de funções:

  • Funções embutidas (de fábrica)

  • Funções definidas pelos usuários (user defined functions - UDFs)

  • Funções anônimas (também conhecidas como funções lambda)

Primeiro, você pode verificar aqui as funções embutidas do Python. Mas, obviamente, o legal é criarmos nossas próprias funções (UDFs)! As funções to tipo lambda ficam para outro tópico separado.

O que é uma função? De forma simplista, é um “mecanismo que operacionaliza ações”. Vamos usar uma analogia primeiro, e ver os detalhes técnicos depois. Imagine que seu sonho é ter usa própria esfirraria, e por isso precisa assar esfihas em grande quantidade - você precisará de um forno-esteira:

Assim, o forno é uma função - recebe um conjunto de entradas (esfirras cruas no exemplo, mas no nosso caso dados, parâmetros, etc), operacionaliza sua ou sua ações (aqui, assar), e retorna algo no fim (esfirras assadas):

Criando funções:¶

Vamos imaginar que você precisa precificar suas esfirras. Você precisa saber pelo menos os custos, impostos e margem de lucro. Se você lembra da matemática do colegial, podemos então dizer que a função preco contém os três: \(preco = f(custos, impostos, margem)\). Assim, usando esses três dados como entrada você automatiza a operação do cálculo (função) e recebe no fim o resultado (saída).

Vamos começar criando a função preco, passo a passo. Primeiro, para criar uma função você usa a palavra reservada def e em seguida o nome da função que você escolheu, seguido de (). Para não termos erro (porque precisa ter algum conteúdo), usamos a palavra reservada pass (que substitui o conteúdo e evita erros):

def preco():
    pass

Agora que temos a função, podemos acrescentar uma funcionalidade - receber um argumento (ou parâmetro). No caso, vamos começar só incluindo o custo da esfirra. Para retornar o resultado no fim, usamos a palavra reservada return:

def preco(custo):
    return custo

Dica: De forma mais técnica, um parâmetro é o nome que damos dentro dos parênteses da função e um argumento é valor atrelado a esse parâmetro. Na prática, usamos como sinônimos.

Vamos testar para ver se funciona:

preco(1.6)

1.6

Bingo! Funciona perfeitamente, apesar de ser bem inútil por enquanto… Vamos agora adicionar impostos! Para simplificar, vamos acrescentar 30% aos custos:

def preco(custo):
    impostos = custo * 0.3
    return custo + impostos

Funciona como esperávamos. Agora, vamos acrescentar a margem de lucro. Que tal 10%? Vamos também arredondar para 2 casas decimais:

def preco(custo):
    impostos = custo * 0.3
    lucro = (custo + impostos) * 0.1
    return round(custo + impostos + lucro, 2)

preco(1.6)

2.29

Claro, você sempre pode reorganizar o código para não criar variáveis intermediárias, mas note que já temos uma função funcionando (piadinha intencional kkk).

Mas e se quisermos alterar a margem de lucro? Podemos criar um novo argumento na função:

def preco(custo, lucro):
    impostos = custo * 0.3
    return round(custo + impostos + (custo + impostos) * lucro, 2)

O que acontece se trocarmos as informações sem querer?

Teremos um retorno numérico (afinal a função sabe calcular), mas sem necessariamente ser o resultado que queremos. Para isso, usamos os argumentos por nome quando fazer as entradas. Agora não tem confusão!

Isto é, ou você obedece a ordem em que os argumentos são colocados dentro da função, ou explicita os argumentos na passagem de valores. (Na verdade você pode iniciar com argumentos posicionais sem referenciar os nomes e depois explicitar na ordem que quiser, mas a maioria das pessoas acha confuso e acaba dando em erro por causa disso…).

Outra coisa a ser observada é que você terá erro se tiver mais ou menos argumentos que os escritos na função:

O que acontece se quisermos deixar como padrão o lucro em 10%, mas podendo alterar conforme a necessidade? Usamos um valor padrão (default) explicitando-o na criação da função:

def preco(custo, lucro=0.1):
    impostos = custo * 0.3
    return round(custo + impostos + (custo + impostos) * lucro, 2)

O texto de documentação é chamado pelo Python quando você usar com uma função de ajuda. Para inserir um texto de documentação em uma função, use um comentário com três aspas logo após a definição da função:

    Função preco: recebe o custo, recebe lucro (por padrão 0.1)

    """

Vale a pena discutir um pequeno detalhe muitas vezes ignorado - a diferença entre print() e return ao fim das funções.

  • print() é uma função, que imprime (dispõe o texto na tela)

  • return é uma palavra reservada, que devolve o valor após return de volta ao código

O que aprendemos até aqui? Funções são muito úteis porque:

  • Permitem abstração (fazer ações em nível alto ao invés de mexer debaixo do capô): depois que você criou a função custo (e testou obviamente) você pode simplesmente pensar entradas -> custo() -> saída).

  • Promovem reusabilidade (executar a tarefa inúmeras vezes com menos trabalho): agora você pode virar o magnata do ramo de comida árabe sem calcular na mão o preço de cada esfirra!

  • São modulares (um pedaço de código que pode ser acoplado a diversas outras partes de código): agora que a função está pronta, pode ser usada em conjunto com outras funções e outros programas seus!

Funções anônimas (lambda)

Python foi pensada originalmente como uma linguagem mais declarativa e menos funcional, mas em algum momento iniciou-se o suporte à programação funcional.

A ideia por trás das funções anônimas (ou funções lambda, também conhecidas como expressões lambda) é serem descartáveis. Ou seja, ao contrário das funções que vimos anteriormente, essas não ficam guardadas na memória.

Vamos ver uma função tradicional e comparar com uma lambda:

def quadrado(i):
    return i * i

quadrado(2)

4

Uma função anônima:

  • É sempre iniciada com a palavra reservada lambda

  • Recebe ao menos uma variável

  • Dois pontos para marcar o início da expressão (:)

  • E tem a operação desejada.

Como num laço do tipo for, o nome da variável é criada dentro da própria expressão e você pode chamar do que quiser (i, x, etc.).

# não tem nome "quadrado"
lambda x: x * x

<function __main__.<lambda>(x)>

A função lambda não é guardada com um nome, portanto só é usada quando chamada e some em seguida (tanto que no exemplo acima, não faz nada). Vamos usar as funções lambda junto com a biblioteca pandas mais à frente, mas para exemplificar, podemos usar com a função map() (vamos ver na aula que vem map() com mais detalhes):

lista = [0, 2, 4, 6, 8]

lista_quadrado = []

for i in lista:
    lista_quadrado.append(quadrado(i))

lista_quadrado

[0, 4, 16, 36, 64]

lista_quadrado = list(map(lambda x: x * x, lista))
lista_quadrado

[0, 4, 16, 36, 64]

PS: É lambda da letra grega \(\Lambda\), não da dança típica lambada, nem lambida 😜

Anotação de tipos

Por mais que Python seja uma linguagem dinâmica, às vezes você pode sentir falta da clareza de declarar tipos de variáveis. Para tanto, você pode usar anotações de tipo (type annotations), o que vai facilitar a leitura de variáveis, por exemplo, em uma função.

Como vimos anteriormente, para declarar uma variável em Python, você simplesmente dá um nome e passa um valor, o interpretador se encarrega de inferir o tipo de variável:

ano = 365
type(ano)

int

Usar anotações de tipo não altera em absolutamente nada o processo, mas faz com que seja mais legível:

ano: int = 365
type(ano)

int

O interpretador simplesmente ignora essa informação extra, e não há impacto positivo (reserva de memória) nem negativo (rendimento mais lento), afinal é somente um tipo de comentário.

Vamos ver um exemplo aqui mais complexo:

def texto_preco_prato(preco: float, unidades: int, nome: str) -> str:
    """
    Retorna uma mensagem com o nome, preco e unidades restantes de um prato.
    """
    print(f"O prato {nome} custa R$ {preco} e só restam {unidades} unidades!")

texto_preco_prato(2.98, 50, "esfirra de queijo")

O prato esfirra de queijo custa R$ 2.98 e só restam 50 unidades!

Espaços de nomes (Namespaces)

Por outro lado, não aprendemos nada ainda sobre namespaces (espaços de nomes). Isso é importante porque dentro de funções sempre estamos manipulando variáveis.

Python por padrão já vem com um espaço de nomes - de onde vêm, por exemplo, as palavras reservadas. Em seguida, temos o espaço de nomes global - onde ficam as variáveis por exemplo que vamos criando (ou as que criamos até aqui). Em seguida, temos o enclosing (“cerco” ou “fechamento”), que são as variáveis dentro de uma função. No exemplo logo há pouco, criamos uma variável impostos dentro de uma função e por isso, ela não está disponível fora da função.

#retorna erro!
#impostos

Por último, temos o local, que é, por exemplo, uma variável, dentro de uma função dentro de outra função. Isso tudo é para você entender que Python não confunde os nomes em cada espaço. Você pode criar uma variável global impostos e quando criar uma função que contém uma variável impostos não haverá conflito, o Python criou outro espaço de nomes. (Note que isso é uma prática ruim, mas você entendeu o exemplo!).

Vamos fazer isso só para exemplificar - criamos uma variável global raio que contém o texto “Cabum!”. Em seguida criamos a função área, que tem um variável local raio que recebe um valor numérico. Se a função usar raio global, o resultado com certeza não será numérico!

raio = "Cabum!"

def area(raio):
    PI = 3.14
    return PI * raio ** 2

area(4)

50.24

Para relembrar - temos quatro namespaces (espaços de nomes) básicos:

  • Embutido (built-in): o que já vem por padrão incluindo palavras reservadas

  • Global (global): o que temos acesso mais fácil como usuários

  • “Cerco” (enclosing): dentro de uma função

  • Local (local): dentro de uma função, dentro de outra função (por exemplo)

Assim a ordem de chamada de um nome ligado a mais de uma variável segue a ordem de dentro para fora (em inglês, usa-se muito o acrônimo LEGB):

Exemplo (tirado daqui):

x = "global" # agora x var ser global (porque criamos no nosso namespace padrão que é o global)

def f():
    x = "enclosing" # agora x vai ser enclosing (porque entramos em outro namespace)
    
    def g():
        x = "local" # agora x vai ser local (porque entramos em mais um namespace)
        print(x) 
    
    g() # imprime o x de g(), ou seja, local
    
    print(x)
    
f() # imprime o x de f(), ou seja, enclosing

print(x) # imprime o x do namespace global, ou seja, global
